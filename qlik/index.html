🚀 Mastering Qlik Sense Upgrade: A Comprehensive Guide

Hello Qlik community! 🌐 Today, we're diving into the intricacies of upgrading your Qlik Sense deployment, ensuring a seamless transition and unlocking the latest features. Below, we elaborate on two expert mode options for your Qlik Sense upgrade journey.

Prerequisites to Upgrade or Install Qlik Sense

Admin User: Ensure admin rights for system updates and Remote Desktop Access for Qlik Sense installation or upgrade.
Access services via: C:\ProgramData\Microsoft\Windows\Start Menu\Programs\Administrative Tools\services.lnk

Ports Required for Qlik Sense Communication

Ports should be open and available for communication in Central/Rim Node:

  • 80: Proxy (HTTP)
  • 443: Proxy (HTTPS)
  • 4243: Qlik Sense Proxy Service (QPS) REST API
  • 4244: Windows authentication
  • 4748: Notifications from QRS
  • 4242: QRS REST API Bi-directional between the central node and all proxy nodes
  • 4239: QPS - WebSocket
  • 4432: Default listen port for database connections
  • 4899: Qlik Sense Printing Service (QPR) port
  • 4432: For connecting to the Qlik Sense Repository Database
  • 4444: Between the central node and all rim nodes
  • 5050 & 5151: Scheduler
  • 4747: Qlik Sense Engine Service (QES) listen port
  • 9028: Default listen port for the Hub Service
  • 4949: Data Profiling Service when accessing and modifying the application load model
  • 4900: Broker Service

License Activation Process

When licensing Qlik Sense using a serial and control number:

  • The LEF can be downloaded when the serial number and the control number have been entered in the Qlik Management Console (QMC).
  • Note: No Whitelist is required.
  • Signed License can be attained by the organization or Qlik sales team.
  • SLD (Signed License Distribution) Link: https://license.qlikcloud.com/sld
  • The SLD link will remain active for 90 days only.
  • Whitelist in the Proxy Exceptions list bypasses license.qlikcloud.com and lef4.qliktech.com.

Backup Plan

  • If on Virtual Machine take snapshot or backup the below folders:
    • C:\ProgramData\Qlik
    • C:\Program Files\Qlik
    • Qlik Share Folder (App/Static Content/Archived Log/etc)
  • Take manual backup for below.
    • The SSL Certificate thumbprint backup from QMC-> Proxies-> Central -> Security.
    • Qlik Repository Database backup.
    • User Directory Path Backup from QMC user sync tasks.
    • All the Published & Non-Published (My Work) App Backup for All the users by doing manually Export.
    • Backup of Monitoring Apps
    • The Security Rule Backup if any Custom Security.
    • All the Whitelist name backup.
