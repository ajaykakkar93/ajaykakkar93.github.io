# Projects — Medly Pharmacy

**Tenure:** Oct 2020 – Oct 2022
**Role:** Qlik Sense Developer
**Location:** Pune, India

---

## Company Context

Medly Pharmacy was a US-based specialty pharmacy platform. Ajay built the analytics infrastructure — dashboards, compliance reports, NPS tracking, and operational reporting — using Qlik Sense, Python, and NPrinting.

---

## Key Achievements

- Pioneered **Python-to-Qlik live data integration** — first-of-kind for the team
- Established **JavaScript-to-Python connectivity** within Qlik Sense for real-time data flow
- Implemented **on-the-fly chart generation** (live data without loading full datasets)
- Built **NPrinting automation** for 20+ report workflows, eliminating manual report distribution
- Delivered end-to-end compliance reporting for URAC, GCA, and regulatory audits

---

## Projects

### Project 1 — Demographics Info Dashboard & Mashup

**Type:** Dashboard + Mashup
**Highlights:**
- Python integration to fetch live data in real time
- JavaScript-to-Python connectivity for Qlik Sense data pipeline
- On-the-fly chart generation: displays live data per parameter without loading the full dataset

### Project 2 — General Compliance Audit 1 & 2

**Type:** Compliance Reporting Dashboard
**KPIs covered:**
- Coupon Opportunity (commercial coupon eligibility analysis)
- Underpaid Medication Claims (claim discrepancy detection)
- Medications Prescribed Repeatedly (prescription pattern monitoring)
- 50 Mile Covered Zip Codes (zip-code-based geographic coverage with miles calculation)

---

### Project 3 — URAC Dashboard & NPrinting Reports

**Type:** Dashboard + Automated Reports
**URAC = Utilization Review Accreditation Commission** (pharmacy compliance standard)
**KPIs covered:**
- Number of specialty patients and prescriptions dispensed
- URAC Net Promoter Score (NPS — patient ratings)
- Call abandonment rate and call center performance metrics
- QRE (Quality-Related Events) details
**Automation:** Reports triggered via NPrinting to business users on schedule

---

### Project 4 — DNA Apps Dashboard

**Type:** Multi-dashboard suite
**Three dashboards built:**
1. **Provider DNA** — provider count, targets, performance metrics
2. **Patient DNA** — patient count, targets, engagement metrics
3. **Retail DNA** — retail count, targets, distribution metrics

**Data pipeline:** Reports developed using Python-generated data stored as CSV, loaded into Qlik

---

### Project 5 — Delighted App (NPS Score) Dashboard & NPrinting

**Type:** Dashboard + Automated Email Reporting
**Delighted** = customer satisfaction platform
**Features:**
