## Current Work — Vendor Enrollment (OCR Feature)

**Problem being solved:**
Vendor enrollment currently requires vendors to manually submit ID cards and other documents, which are then manually re-entered by finance/operations staff into the system — a slow, error-prone process.

**What Ajay is building:**
An OCR-powered document scanning and validation pipeline for Indian vendor KYC documents:

**Documents supported:**
- **PAN Card** — Permanent Account Number (tax identity, mandatory for Indian businesses)
- **Aadhaar Card** — National biometric ID (address + identity proof)

**How it works:**
1. Vendor submits ID card image during enrollment
2. OCR extracts structured fields automatically — name, ID number, DOB, address
3. System validates the document (checks format, required fields, authenticity signals)
4. Only validated vendors proceed to enrollment — invalid/incomplete docs are flagged
5. Extracted data auto-populates the enrollment form — zero manual re-entry

**UI/UX impact (equally important):**
- Dramatically fewer steps in the enrollment journey
- Less manual data entry = fewer errors, faster completion
- Smoother, modern user experience — vendor onboarding feels effortless
- Consistent with KADAC's philosophy of removing friction from finance operations

**Tech involved:** OCR, AI document processing, Indian KYC document validation, form auto-population, S2P workflow integration

**Impact:** Turns a slow, error-prone, manual KYC process into a validated, automated pipeline — reducing enrollment time significantly and improving data accuracy.
